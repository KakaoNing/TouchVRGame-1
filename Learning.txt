3. 몇몇 렌더기능 세팅으로 최적화를 먼저 시도한다 (영어검색 안되면 한글)

3-1 Forward Shading - 포워드 쉐이딩 활성화 : 사용하면 VR에서는 사용하지않는 기능을 비활성화하여 조금더 빠르게 렌더링이 가능하다 이를통해 VR에 필요한 높은 렌더 해상도와 높은 프레임레이트를 유지 가능하다.
3-2 Motion Blur - 모션블러 비활성화 : 포워드 렌더는 모션블러를 지원하지않는다 비활성화하는게 좋은습관이다 게다가 모션블러는 VR에서 편리한 기능이아니다 카메라 또는 사용자의 머리가 움직일때마다 사용자의 뷰가 블러처리되기 때문
3-3 Instanced Stereo Rendering - 인스턴스 스테레오 렌더링 활성화 : 두 카메라가 렌더 데이터의 특정 영역을 공유하여 드로 스레드 주기를 단축한다
3-4 Anti-Aliasing Method - MSAA설정하기 : MSAA는 포워드 렌더링 작업이 가능하며 대부분의 GPU에서 좋은 퀄리티를 제공한다. MSAA는 GPU의 메모리를 활용하여 스무딩 효과를 연출한다 메모리 스펙이 낮은 글카의 경우 퍼포먼스의 영향

4. 지원 플랫폼 설정 : 프로젝트 섹션에서 퍼블리시할 플랫폼 사용할것만 설정 (우리는 퀘스트로 개발할 것이기 때문에 Android도 포함해야한다

5. VR에서 각 게임플레이 세션을 자동으로 시작하려면 - 설명 카테고리에서 Start in VR 활성화 Smooth Frame Rate 비활성화 : 이 옵션은 타깃 하드웨어와 FPS가변하는 상황에서는 도움이 됩니다. 그러나 우리는 고정 프레임 레이트를 확보하고자 하는 상황이기때문에 이기능은 유용하지 않다

6. Use Fixed Frame Rate 비활성화 - 고정프레임 : 프레임고정 하드웨어가 해당 FPS를 얻지 못하면 FPS는 게임의 클록 속도를 낮춰 프레임레이트를 맞춘다

7. Custom Timestamp 를 None으로 설정한다 : 커스텀 타임스탬프는 프레임 단위의 시간이 중요한 어플리케이션에 유용하다 VR에서는 필요없다

8. Quest VR 로 개발중이라면 Mobile HDR 비활성화, Mobile Multi-View 활성화 : 이렇게 해도 Rift에는 영향을 미치지않는다
